## Размер модулей и декомпозиция

| Размер файла | Правило |
|---:|---|
| до `250` строк | Нормально. |
| `250–400` строк | Проверить, не смешаны ли зоны ответственности. |
| `400+` строк | Перед новыми фичами рассмотреть декомпозицию. |
| `600+` строк | Нужен рефакторинг до новых крупных правок. |

Правила:

1. Один файл = одна зона ответственности.
2. Не делать hotfix-файлы.
3. Не писать бизнес-логику в `index.html`.
4. Постоянные стили — только в CSS.
5. CSS держать модульно: отдельный экран/слой получает отдельный файл.
6. Новые файлы создавать только если они убирают смешение ответственности.
7. После изменения структуры обновлять этот документ.

## Жёсткие правила гонки

- Phaser отвечает только за трассу, лошадей, камеру и эффекты.
- HUD гонки рисуется HTML/CSS overlay поверх canvas.
- Не возвращать текст HUD в Phaser без веской причины.
- Вид гонки — строго сверху.
- Текущая стабильная форма трассы: реалистичный овал.
- S-трасса — будущий отдельный прототип, не поверх рабочей гонки.
- Миникарта не используется: прогресс показывается верхней линией.
- Лошади стартуют с одной линии.
- Внешние дорожки имеют большую реальную длину через `laneDistanceFactor`.
- Искусственный штраф внешней дорожки не используется.

## Лошадь и класс

- Основные параметры: `speed`, `stamina`, `acceleration`.
- Класс считается только по ним:
  - скорость — `40%`;
  - выносливость — `35%`;
  - ускорение — `25%`.
- Качества не входят напрямую в класс.
- Качества влияют на события и поведение в гонке:
  - сила;
  - ловкость;
  - чутьё.

## Физика выносливости

- Выносливость работает как бак.
- `staminaTankMax = effectiveStamina * 35`.
- `drainPerSecond = currentSpeedKmh`.
- Рывок увеличивает расход: `drainPerSecond *= 2`.
- Плохая траектория не увеличивает расход выносливости.
- Остаток бака влияет на скорость через `staminaSpeedFactor`.

## Баланс заездов

- Сложность заезда не масштабируется от лошади игрока.
- У каждого заезда фиксированный диапазон ботов.
- Боты создаются в `race-menu.js` по параметрам выбранного заезда.
- Текущая сетка:
  - Новичковый: `22–35`, `150` м;
  - Обычный: `35–50`, `200` м;
  - Сильный: `50–70`, `300` м;
  - Элитный: `70–90`, `500` м.

## Карточка лошади

- Единый рендер: `SKACHKI_HORSE_UI.renderHorseCard`.
- JS карточки: `js/horse-ui.js`.
- SVG-портрет: `js/horse-portrait.js`.
- CSS карточки: `css/horse-card.css`.
- Потенциал показывается звёздами.
- Класс — цифра на медальоне, без слова `Класс`.

## Разведение

- Пользователь выбирает жеребца и кобылу.
- Если своих родителей нет — доступны сервисные родители за монеты.
- Сервисные родители не сохраняются в Конюшню.
- Потомок наследует породу, масть, характер, показатели, качества и потенциал.
- `Будущий жеребёнок` показывает прогноз, не точные значения.

## События гонки

- События не должны быть чистым таймером + случайностью.
- Базовая оценка: каждые `20` метров.
- Учитываются фаза дистанции, поворот/прямая, выносливость, скорость, ускорение, качества, форма и соперники рядом.
- События первой версии:
  - `burst`;
  - `mistake`;
  - `fatigue`;
  - `strong_finish`.
- Визуал событий находится в `js/race-effects.js`.

## Telegram Mini App

- Верх не должен перекрываться интерфейсом Telegram.
- Основная навигация — нижнее меню.
- Важные кнопки не должны попадать под нижнее меню.
- Длинные экраны должны скроллиться.

## Проверка после изменений

Проверить:

- Конюшня, Тренировка, Разведение, Скачки, Рейтинг открываются;
- гонка запускается;
- DOM HUD отображается поверх canvas;
- текст HUD стал чётче;
- кнопка звука работает;
- лидерборд обновляется;
- нижний HUD показывает позицию, скорость и выносливость;
- верхний прогресс обновляется;
- стадионный декор не мешает обзору;
- лошади плавно перестраиваются;
- результаты гонки открываются;
- Telegram UI не перекрывает важные элементы.
