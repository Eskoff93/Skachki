# Архитектура проекта Skachki

## Текущее состояние ветки `cleanup`

Проект уже переведён с набора временных patch-файлов на более чистую структуру.

### Основные файлы

- `index.html` — HTML-разметка экранов и подключение скриптов.
- `app.js` — текущий основной игровой файл. Пока содержит большую часть логики.
- `css/app.css` — общий CSS приложения, подключает базовый `style.css`.

### Модули в `js/`

- `js/utils.js` — общие утилиты.
- `js/data.js` — статические игровые данные.
- `js/horse.js` — доменная логика лошади.
- `js/state.js` — сохранение, загрузка и сброс прогресса.

## Почему `app.js` пока не разбит полностью

`app.js` большой и содержит смешанную логику:

- состояние игры;
- UI;
- конюшня;
- тренировки;
- разведение;
- меню гонок;
- Phaser-гонка;
- результаты;
- Telegram Web App настройки.

Из-за размера файла его нельзя безопасно править частями без полного контроля содержимого. Поэтому разбиение нужно делать постепенно и проверять игру после каждого этапа.

## Порядок дальнейшего cleanup

### Этап 1 — данные

Перевести `app.js` на использование `window.SKACHKI_DATA`:

- `stableNames`;
- `temperaments`;
- `trainingTypes`;
- `raceTypes`;
- `botNames`;
- `parameterHelp`.

### Этап 2 — доменная логика лошади

Перевести `app.js` на использование `window.SKACHKI_HORSE`:

- `createHorse`;
- `horseClass`;
- `behaviorLabel`.

### Этап 3 — сохранение

Перевести `app.js` на использование `window.SKACHKI_STATE`:

- `saveGame`;
- `loadGame`;
- `reset`.

### Этап 4 — UI

Вынести из `app.js`:

- главное меню;
- конюшню;
- тренировки;
- разведение;
- меню гонок;
- результаты.

### Этап 5 — гонка

Отдельно переработать Phaser-сцену:

- `race-engine.js`;
- боковая таблица лидеров;
- события над лошадьми;
- баланс времени гонки;
- вид сверху.

## Правило разработки

Не добавлять новые hotfix-файлы поверх `app.js`.

Все новые изменения должны идти либо:

1. в существующие модули `js/`,
2. либо через осознанное уменьшение `app.js`.

После каждого этапа обязательно проверять игру по GitHub Pages.
