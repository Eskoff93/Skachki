## Правила структуры

- Один файл = одна зона ответственности.
- Не делать hotfix-файлы.
- Не писать бизнес-логику в `index.html`.
- Постоянные стили — только в CSS.
- CSS держать модульно.
- После изменения структуры обновлять этот документ.

## Жёсткие правила гонки

- Phaser отвечает только за трассу, лошадей, камеру и эффекты.
- HUD гонки рисуется HTML/CSS overlay поверх canvas.
- Не возвращать текст HUD в Phaser без веской причины.
- Вид гонки — строго сверху.
- Текущая стабильная форма трассы: реалистичный овал.
- S-трасса — будущий отдельный прототип, не поверх рабочей гонки.
- Миникарта не используется: прогресс показывается верхней линией.
- Лошади стартуют с одной линии.
- Внешние дорожки имеют большую реальную длину через `laneDistanceFactor`.
- Искусственный штраф внешней дорожки не используется.

## Чистый баланс-тест

- Включается в `js/balance-test.js` переключателем `Чистый тест параметров`.
- Передаёт флаг `pureBalanceTest` в `activeRaceType`.
- В чистом режиме работают только:
  - скорость;
  - выносливость;
  - ускорение;
  - дистанция.
- В чистом режиме отключаются:
  - форма и случайные form ticks;
  - randomFactor;
  - AI-перестроения;
  - события гонки;
  - рывки;
  - сбои;
  - усталостные события;
  - дорожечный коэффициент `laneDistanceFactor`;
  - визуальная пыль от рывков.
- Лошади остаются на стартовых дорожках, дистанция для всех одинаковая.
- Этот режим нужен только для проверки баланса формул, без наград и статистики.

## Лошадь и класс

- Основные параметры: `speed`, `stamina`, `acceleration`.
- Класс считается только по ним:
  - скорость — `40%`;
  - выносливость — `35%`;
  - ускорение — `25%`.
- Качества не входят напрямую в класс.
- Качества влияют на события и поведение в гонке.

## Физика выносливости

- Выносливость работает как бак.
- `staminaTankMax = effectiveStamina * 35`.
- `drainPerSecond = currentSpeedKmh`.
- Рывок увеличивает расход: `drainPerSecond *= 2`.
- Плохая траектория не увеличивает расход выносливости.
- Остаток бака влияет на скорость через `staminaSpeedFactor`.

## Баланс заездов

- Сложность заезда не масштабируется от лошади игрока.
- У каждого заезда фиксированный диапазон ботов.
- Боты создаются в `race-menu.js` по параметрам выбранного заезда.
- Текущая сетка:
  - Новичковый: `22–35`, `150` м;
  - Обычный: `35–50`, `200` м;
  - Сильный: `50–70`, `300` м;
  - Элитный: `70–90`, `500` м.

## События гонки

- События не должны быть чистым таймером + случайностью.
- Базовая оценка: каждые `20` метров.
- Визуал событий находится в `js/race-effects.js`.

## Проверка после изменений

Проверить:

- гонка запускается;
- DOM HUD отображается поверх canvas;
- обычный баланс-тест работает как раньше;
- чистый баланс-тест отключает форму, AI, события, рывки и дорожечные коэффициенты;
- результаты чистого теста стабильны при повторных запусках;
- верхний прогресс обновляется;
- результаты гонки открываются;
- Telegram UI не перекрывает важные элементы.
