# Архитектура проекта Skachki

## Текущее состояние ветки `cleanup`

Проект переведён с большого монолитного `app.js` и временных patch/hotfix-файлов на модульную структуру.

`index.html` больше не подключает `app.js`. Запуск приложения идёт через `js/bootstrap.js`.

## Основные файлы

- `index.html` — HTML-разметка экранов и порядок подключения модулей.
- `css/app.css` — общий CSS приложения, подключает базовый `style.css`.
- `phaser.min.js` — библиотека Phaser.
- `horse_icon.png` — изображение лошади.

## Модули в `js/`

- `js/bootstrap.js` — запуск приложения и Telegram Web App init.
- `js/game.js` — общее состояние игры, сохранение, базовые хелперы, переключение экранов.
- `js/data.js` — статические игровые данные: заезды, тренировки, имена ботов, подсказки.
- `js/horse.js` — доменная логика лошади: создание, класс, описание характера.
- `js/state.js` — сохранение, загрузка и сброс прогресса.
- `js/ui.js` — общие UI-хелперы: поиск DOM-элементов, toast, переключение экранов.
- `js/stable.js` — главное меню, конюшня, карточки лошадей, подробности.
- `js/training.js` — экран тренировок и логика прокачки.
- `js/breeding.js` — экран разведения и создание потомков.
- `js/race-menu.js` — выбор типа гонки, выбор своей лошади, генерация соперников.
- `js/race-engine.js` — Phaser-гонка, трек, движение лошадей, события гонки, таблица лидеров.
- `js/results.js` — результаты гонки, награды, списание энергии.
- `js/utils.js` — небольшие общие утилиты.

## Правила дальнейшей разработки

1. Не добавлять новые hotfix-файлы поверх основной логики.
2. Все изменения вносить в соответствующий модуль.
3. Если меняется гонка — править `js/race-engine.js`.
4. Если меняется экономика заездов — править `js/data.js` и `js/results.js`.
5. Если меняется конюшня — править `js/stable.js`.
6. Если меняется тренировка — править `js/training.js`.
7. Если меняется разведение — править `js/breeding.js`.
8. После каждого изменения проверять GitHub Pages и Telegram Mini App.

## Что ещё осталось сделать

- Удалить старый `app.js`, если он ещё физически лежит в репозитории.
- Проверить все экраны после удаления.
- После этого можно переходить к балансу и игровому циклу.
