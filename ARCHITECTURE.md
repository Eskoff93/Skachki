# Архитектура проекта Skachki

`ARCHITECTURE.md` — источник правды по структуре проекта.

Перед крупными изменениями нужно сверяться с этим файлом. Если фактическая структура проекта или договорённости устарели, сначала нужно явно сказать об этом и обновить `ARCHITECTURE.md`, затем вносить кодовые изменения.

## Основные правила

1. Не добавлять hotfix-файлы поверх основной логики.
2. Не размещать игровую бизнес-логику в `index.html`.
3. Все изменения вносить в соответствующий модуль.
4. Сложные изменения делать небольшими безопасными шагами.
5. Не использовать костыли, если можно исправить архитектурно правильно.
6. Не создавать лишние файлы без необходимости.
7. Если новый файл уменьшает сложность и убирает смешение ответственности — его можно создать.
8. После изменения структуры проекта обновлять этот файл.
9. После каждого изменения проверять GitHub Pages и Telegram Mini App.

## Назначение ключевых модулей

### `index.html`

Только базовая разметка экранов и порядок подключения CSS/JS. Не содержит бизнес-логику, расчёты, сложный рендер или большие inline-стили.

### `css/app.css`

Базовый layout, экраны, кнопки, модальные окна и общие компоненты.

### `css/luxury.css`

Премиальный визуальный слой: карточки лошадей, медальоны, нижнее меню, Разведение, экран рождения жеребёнка и редактор имени.

### `js/game.js`

Центральный координатор: состояние, сохранение, переходы между экранами и общие helper-функции.

### `js/horse.js`

Доменная логика лошади: создание, нормализация, класс, форма, пол, скрытые качества и ранги.

### `js/ui.js`

Общие UI-компоненты. Единый источник карточки лошади — `SKACHKI_HORSE_UI.renderHorseCard`.

### `js/breeding-service.js`

Модуль сервисных родителей. Отвечает за племенного жеребца, племенную кобылу, цены, доступность и поиск родителей.

Статус: создан, подключён, используется через `SKACHKI_BREEDING_SERVICE`.

### `js/breeding-logic.js`

Модуль расчётов Разведения. Отвечает за наследование статов, скрытых качеств, потенциала, прогноз потомка и создание жеребёнка.

Статус: создан и подключён перед `js/breeding.js`. Для завершения шага нужно загрузить обновлённый `js/breeding.js`, который вызывает `SKACHKI_BREEDING_LOGIC`.

### `js/breeding.js`

Координатор Разведения. Должен отвечать за выбранных родителей, шаги экрана, события и вызовы service/logic/render.

Текущий статус: сервисная логика уже вынесена. Расчёты наследования подготовлены к выносу в `js/breeding-logic.js`, но нужна ручная замена `js/breeding.js`.

### `js/breeding-render.js`

Промежуточный модуль визуальных улучшений Разведения. Сейчас отвечает за карточку рождённого жеребёнка и редактор имени. Позже сюда переносится HTML-рендер Разведения.

## Правила карточки лошади

1. Потенциал отображается звёздами: 50–60 одна, 60–70 две, 70–80 три, 80–90 четыре, 90–100 пять.
2. Класс отображается цифрой на медальоне.
3. Пол отображается значком.
4. Слова `Класс` на медальоне быть не должно.
5. Карточка должна совпадать в Конюшне, выборе лошади в Гонках и выборе родителей в Разведении.

## Правила Разведения

1. Пользователь выбирает жеребца и кобылу.
2. Если своих доступных жеребцов нет, доступен `Племенной жеребец`.
3. Если своих доступных кобыл нет, доступна `Племенная кобыла`.
4. Сервисные родители не сохраняются в Конюшню и не получают `offspringCount`.
5. Цена: жеребец — `80`, кобыла — `100`.
6. Потомок наследует породу, масть, характер, обычные показатели, скрытые качества и потенциал.
7. Блок `Будущий жеребёнок` показывает прогноз, точные значения открываются после рождения.
8. У рождённого жеребёнка должна быть возможность задать имя.

## Правила Telegram Mini App

1. Верх интерфейса не должен перекрываться системным UI Telegram Mini App.
2. Основная навигация — нижнее меню.
3. Важные кнопки не должны прятаться под нижним меню.
4. Скролл должен работать на всех длинных экранах.

## Порядок безопасного рефакторинга

1. Вынести inline JS из `index.html`. Статус: сделано.
2. Перенести стили Разведения из `index.html` и JS в CSS. Статус: сделано.
3. Создать и подключить `breeding-service.js`. Статус: сделано.
4. Перевести `breeding.js` на `SKACHKI_BREEDING_SERVICE`. Статус: сделано.
5. Создать и подключить `breeding-logic.js`. Статус: сделано.
6. Загрузить обновлённый `breeding.js`, который использует `SKACHKI_BREEDING_LOGIC`. Статус: требуется ручная загрузка файла.
7. Перенести HTML-рендер из `breeding.js` в `breeding-render.js`.
8. Уменьшить `breeding.js` до координатора.

## Проверка после изменений в Разведении

Проверить: экран Разведения открывается; есть доступный жеребец и кобыла; работают племенные родители; цена считается корректно; кнопка `Скрестить` активируется корректно; жеребёнок создаётся; имя редактируется; жеребёнок появляется в Конюшне; сервисные родители не появляются в Конюшне; Конюшня, Гонки и Рейтинг не ломаются.
