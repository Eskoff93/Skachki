## Размер модулей и декомпозиция

| Размер файла | Правило |
|---:|---|
| до `250` строк | Нормально. Можно править точечно. |
| `250–400` строк | Осторожно. Перед правкой проверить, не смешаны ли зоны ответственности. |
| `400+` строк | Перед новыми фичами рассмотреть декомпозицию. |
| `600+` строк | Проблемный файл. Нужен рефакторинг до новых крупных правок. |

Правила:

1. Один файл = одна зона ответственности.
2. Не наращивать файл-комбайн, если можно вынести самостоятельный модуль.
3. Не делать обходные патчи через `bootstrap.js`, если проблема находится в конкретном модуле.
4. Перед крупной правкой проверять количество строк и ответственность файла.
5. Новые файлы создавать только если они уменьшают смешение ответственности.
6. После изменения структуры обновлять `ARCHITECTURE.md`.

## Жёсткие правила

1. Не добавлять hotfix-файлы.
2. Не писать бизнес-логику в `index.html`.
3. Не дублировать карточку лошади. Использовать `SKACHKI_HORSE_UI.renderHorseCard`.
4. Не дублировать сервисных родителей вне `breeding-service.js`.
5. Не дублировать расчёты наследования вне `breeding-logic.js`.
6. Постоянные стили — только в CSS, не через JS `<style>`.
7. CSS держать модульно: новый экран получает отдельный файл в `css/`.
8. Новые файлы создавать только если они убирают смешение ответственности.
9. После изменения структуры обновлять этот документ.
10. Не дробить мелкие функции в отдельные файлы без реальной пользы.
11. Перед правкой файла `400+` строк сначала предложить план декомпозиции.
12. В публичный репозиторий добавлять только аудио/изображения с правами использования.

## Карточка лошади

- Единый рендер: `SKACHKI_HORSE_UI.renderHorseCard`.
- JS карточки находится в `js/horse-ui.js`.
- SVG-портрет находится в `js/horse-portrait.js`.
- Стили карточки находятся в `css/horse-card.css`.
- Потенциал: звёзды.
  - 50–60: 1
  - 60–70: 2
  - 70–80: 3
  - 80–90: 4
  - 90–100: 5
- Класс: цифра на медальоне, без слова `Класс`.
- Пол: значок.
- Одна карточка используется в Конюшне, Гонках, Разведении и на экране рождения.
- Экран рождения отличается только анимацией и подачей.

## Тренировка

- Логика тренировки находится в `js/training.js`.
- Стили тренировки находятся в `css/training.css`.
- Потенциал не тратится тренировками.
- Чем ближе показатель к природному пределу, тем ниже эффективность и выше стоимость.

## Разведение

- Пользователь выбирает жеребца и кобылу.
- Если своих доступных жеребцов нет — `Племенной жеребец` за `80`.
- Если своих доступных кобыл нет — `Племенная кобыла` за `100`.
- Сервисные родители не сохраняются в Конюшню и не получают `offspringCount`.
- Потомок наследует породу, масть, характер, показатели, скрытые качества и потенциал.
- `Будущий жеребёнок` показывает прогноз, не точные значения.
- Точные значения открываются после рождения.
- Имя жеребёнка можно изменить на экране рождения.

## Гонка

- Вид строго сверху.
- Миникарта не используется: прогресс гонки показывается верхней линией под заголовком ипподрома.
- `race-engine.js` не должен становиться комбайном. Вся отрисовка вынесена в специализированные модули.
- `race-events.js` отвечает за контекстную логику событий и не должен рисовать эффекты напрямую.
- Золотые круги/ореолы вокруг лошади игрока не используются.
- События гонки показываются визуально, без отдельного текстового окна событий.
- Лошадь игрока выделяется аккуратной синей меткой `★` и фокусом камеры.
- HUD должен помещаться на мобильном экране: компактный верх, компактный лидерборд, нижняя панель без перекрытия гонки.

## События гонки

- События больше не должны быть чистым таймером + случайным броском.
- Базовая частота оценки: каждые `20` метров дистанции.
- Контекст учитывает:
  - фазу дистанции: старт, середина, финиш;
  - участок трассы: прямая или поворот;
  - запас выносливости;
  - скорость;
  - ускорение;
  - ловкость;
  - силу;
  - чутьё;
  - форму;
  - соперников рядом и впереди.
- Поддерживаемые события первой версии:
  - `burst` — рывок;
  - `mistake` — сбой;
  - `fatigue` — усталость;
  - `strong_finish` — сильный финиш.
- События должны иметь cooldown и не спамиться.
- Визуал событий находится только в `js/race-effects.js`.

## Музыка

- Фоновая музыка управляется только через `js/music.js`.
- Музыка стартует только после первого действия игрока из-за правил мобильных браузеров.
- Настройки сохраняются в `localStorage`.
- Управление находится на экране `Рейтинг`.
- Поддерживаются включение/выключение и громкость `0–100%`.
- Основной путь файла: `assets/audio/background-music.mp3`.
- В публичный репозиторий нельзя добавлять коммерческую музыку без прав/лицензии.

## Telegram Mini App

- Верх не должен перекрываться интерфейсом Telegram.
- Основная навигация — нижнее меню.
- Важные кнопки не должны попадать под нижнее меню.
- Длинные экраны должны скроллиться.

## Проверка после изменений

После правок проверить:

- Конюшня открывается;
- карточки лошадей отображаются;
- Тренировка открывается;
- Разведение открывается;
- экран рождения работает;
- Скачки открываются;
- Баланс-тест открывается;
- сама гонка запускается;
- HUD гонки помещается на экране;
- золотые круги вокруг лошади игрока отсутствуют;
- события гонки не спамятся;
- рывки чаще видны на прямой и при запасе выносливости;
- сбои чаще видны на поворотах/в группе;
- сильный финиш появляется только ближе к концу;
- Рейтинг открывается;
- блок музыки отображается в Рейтинге;
- кнопка музыки включает/выключает звук;
- ползунок меняет громкость;
- нижнее меню работает;
- Telegram UI не перекрывает важные элементы.
