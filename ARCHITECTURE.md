## Размер модулей и декомпозиция

| Размер файла | Правило |
|---:|---|
| до `250` строк | Нормально. Можно править точечно. |
| `250–400` строк | Осторожно. Перед правкой проверить, не смешаны ли зоны ответственности. |
| `400+` строк | Перед новыми фичами рассмотреть декомпозицию. |
| `600+` строк | Проблемный файл. Нужен рефакторинг до новых крупных правок. |

Правила:

1. Один файл = одна зона ответственности.
2. Не наращивать файл-комбайн, если можно вынести самостоятельный модуль.
3. Не делать обходные патчи через `bootstrap.js`, если проблема находится в конкретном модуле.
4. Перед крупной правкой проверять количество строк и ответственность файла.
5. Новые файлы создавать только если они уменьшают смешение ответственности.
6. После изменения структуры обновлять `ARCHITECTURE.md`.

## Жёсткие правила

1. Не добавлять hotfix-файлы.
2. Не писать бизнес-логику в `index.html`.
3. Не дублировать карточку лошади. Использовать `SKACHKI_HORSE_UI.renderHorseCard`.
4. Не дублировать сервисных родителей вне `breeding-service.js`.
5. Не дублировать расчёты наследования вне `breeding-logic.js`.
6. Постоянные стили — только в CSS, не через JS `<style>`.
7. CSS держать модульно: новый экран получает отдельный файл в `css/`.
8. Новые файлы создавать только если они убирают смешение ответственности.
9. После изменения структуры обновлять этот документ.
10. Не дробить мелкие функции в отдельные файлы без реальной пользы.
11. Перед правкой файла `400+` строк сначала предложить план декомпозиции.
12. В публичный репозиторий добавлять только аудио/изображения с правами использования.

## Лошадь и класс

- Основные гоночные параметры: `speed`, `stamina`, `acceleration`.
- Класс/гоночная сила считается только по основным параметрам:
  - скорость — `40%`;
  - выносливость — `35%`;
  - ускорение — `25%`.
- Качества не входят напрямую в класс.
- Качества влияют на поведение в гонке:
  - `strength` / сила — борьба, усталость, давление группы;
  - `agility` / ловкость — повороты, смена дорожки, риск сбоя;
  - `instinct` / чутьё — момент рывка, сильный финиш, избежание ошибок.

## Физика выносливости

- Выносливость работает как бак.
- Объём бака:
  - `staminaTankMax = effectiveStamina * 35`.
- Расход бака:
  - `drainPerSecond = currentSpeedKmh`.
- Рывок увеличивает расход:
  - `drainPerSecond *= 2`.
- Плохая траектория не увеличивает расход выносливости: внешняя траектория и так хуже из-за пути/позиции.
- Остаток для HUD и результатов:
  - `staminaReserve = staminaTank / staminaTankMax * 100`.
- Остаток бака влияет на скорость через `staminaSpeedFactor`.

## Баланс заездов

- Сложность заезда не должна масштабироваться от лошади игрока.
- У каждого заезда фиксированный диапазон ботов:
  - `botClassMin`;
  - `botClassMax`;
  - `botSpread`;
  - `botQualityMin`;
  - `botQualityMax`.
- Боты создаются в `race-menu.js` по параметрам выбранного заезда.
- Игрок сам выбирает риск: сильная лошадь может перерасти новичковые и обычные заезды.
- Текущая сетка:
  - Новичковый: класс ботов `22–35`, дистанция `150` м;
  - Обычный: класс ботов `35–50`, дистанция `200` м;
  - Сильный: класс ботов `50–70`, дистанция `300` м;
  - Элитный: класс ботов `70–90`, дистанция `500` м.

## Баланс-тест

- Баланс-тест не даёт наград, не тратит карьеру и не сохраняет тестовых лошадей.
- Поддерживает дистанции `150 / 200 / 300 / 500` м и ручную дистанцию.
- Поддерживает `2–8` участников.
- У каждой тестовой лошади можно вручную менять скорость, выносливость, ускорение и форму.
- Пресеты лошадей:
  - слабая;
  - средняя;
  - сильная;
  - топовая;
  - высокая скорость / низкая выносливость;
  - низкая скорость / высокая выносливость.

## Карточка лошади

- Единый рендер: `SKACHKI_HORSE_UI.renderHorseCard`.
- JS карточки находится в `js/horse-ui.js`.
- SVG-портрет находится в `js/horse-portrait.js`.
- Стили карточки находятся в `css/horse-card.css`.
- Потенциал: звёзды.
  - 50–60: 1
  - 60–70: 2
  - 70–80: 3
  - 80–90: 4
  - 90–100: 5
- Класс: цифра на медальоне, без слова `Класс`.
- Пол: значок.
- Одна карточка используется в Конюшне, Скачках, Разведении и на экране рождения.

## Тренировка

- Логика тренировки находится в `js/training.js`.
- Стили тренировки находятся в `css/training.css`.
- Потенциал не тратится тренировками.
- Чем ближе показатель к природному пределу, тем ниже эффективность и выше стоимость.

## Разведение

- Пользователь выбирает жеребца и кобылу.
- Если своих доступных жеребцов нет — `Племенной жеребец` за `80`.
- Если своих доступных кобыл нет — `Племенная кобыла` за `100`.
- Сервисные родители не сохраняются в Конюшню и не получают `offspringCount`.
- Потомок наследует породу, масть, характер, показатели, скрытые качества и потенциал.
- `Будущий жеребёнок` показывает прогноз, не точные значения.
- Точные значения открываются после рождения.
- Имя жеребёнка можно изменить на экране рождения.

## Гонка

- Вид строго сверху.
- Текущая стабильная форма трассы: реалистичный овал.
- S-образная стадионная трасса выбрана как будущий формат, но внедряется только после отдельного безопасного прототипа.
- Миникарта не используется: прогресс гонки показывается верхней линией.
- Лошади стартуют с одной линии.
- Внешние дорожки имеют большую реальную длину через `laneDistanceFactor`.
- Искусственный штраф внешней дорожки не используется.
- `race-engine.js` не должен становиться комбайном.
- `race-events.js` отвечает за контекстную логику событий и не рисует эффекты напрямую.
- События показываются визуально, без отдельного текстового окна.
- Лошадь игрока выделяется синей меткой `★` и фокусом камеры.
- HUD должен помещаться на мобильном экране.

## Будущая S-трасса

- Делается отдельным прототипом, а не поверх рабочей гонки.
- Старт/финиш: сверху на главной прямой.
- Трибуны: главная сверху + небольшие боковые секции.
- Дорожки должны иметь корректные sampled paths.
- Перед заменой овала нужно отдельно проверить:
  - замкнутость каждой дорожки;
  - совпадение стартовой линии;
  - корректный `pointOnTrack`;
  - одинаковую ориентацию спрайтов;
  - корректную камеру;
  - корректный финиш.

## События гонки

- События не должны быть чистым таймером + случайным броском.
- Базовая частота оценки: каждые `20` метров дистанции.
- Контекст учитывает:
  - фазу дистанции;
  - прямую или поворот;
  - запас выносливости;
  - скорость;
  - ускорение;
  - ловкость;
  - силу;
  - чутьё;
  - форму;
  - соперников рядом и впереди.
- Поддерживаемые события первой версии:
  - `burst` — рывок;
  - `mistake` — сбой;
  - `fatigue` — усталость;
  - `strong_finish` — сильный финиш.
- События имеют cooldown и не спамятся.
- Визуал событий находится только в `js/race-effects.js`.

## Музыка

- Фоновая музыка управляется только через `js/music.js`.
- Музыка стартует только после первого действия игрока.
- Настройки сохраняются в `localStorage`.
- Управление находится на экране `Рейтинг`.
- Основной путь файла: `assets/audio/background-music.mp3`.
- В публичный репозиторий нельзя добавлять коммерческую музыку без прав/лицензии.

## Telegram Mini App

- Верх не должен перекрываться интерфейсом Telegram.
- Основная навигация — нижнее меню.
- Важные кнопки не должны попадать под нижнее меню.
- Длинные экраны должны скроллиться.

## Проверка после изменений

После правок проверить:

- Конюшня открывается;
- карточки лошадей отображаются;
- Тренировка открывается;
- Разведение открывается;
- экран рождения работает;
- Скачки открываются;
- заезды показывают фиксированный класс ботов;
- боты не масштабируются от лошади игрока;
- боты получают `hiddenQualities`;
- Баланс-тест открывается;
- пресеты баланс-теста применяются к выбранной тестовой лошади;
- сама гонка запускается;
- стабильный овал отображается корректно;
- старт/финиш находится на прямой;
- внешняя дорожка не получает искусственный штраф скорости;
- внешняя дорожка имеет большую дистанцию через `laneDistanceFactor`;
- бак выносливости тратится от текущей скорости;
- быстрые лошади с низкой выносливостью сильнее проседают на длинной дистанции;
- класс меняется только от скорости/выносливости/ускорения;
- качества не меняют класс напрямую;
- качества влияют на события гонки;
- HUD гонки помещается на экране;
- события гонки не спамятся;
- Рейтинг открывается;
- музыка управляется из Рейтинга;
- нижнее меню работает;
- Telegram UI не перекрывает важные элементы.
