## Ответственность файлов

| Файл | Ответственность |
|---|---|
| `index.html` | Только разметка экранов и подключение CSS/JS. Без бизнес-логики, расчётов, сложного рендера и inline CSS/JS. |
| `css/app.css` | Базовый layout, экраны, кнопки, модалки, общие компоненты. |
| `css/luxury.css` | Визуальный стиль: карточки, медальоны, нижнее меню, Разведение, экран рождения. |
| `js/bootstrap.js` | Запуск приложения. |
| `js/game.js` | Центральное состояние, сохранение, переходы экранов, общие helper-функции. |
| `js/data.js` | Справочники и баланс-данные. |
| `js/horse.js` | Доменная логика лошади: создание, нормализация, класс, пол, форма, ранги, скрытые качества. |
| `js/state.js` | `localStorage`, миграции, сериализация. |
| `js/ui.js` | Общие UI-компоненты. Главный источник карточки: `SKACHKI_HORSE_UI.renderHorseCard`. |
| `js/stable.js` | Экран Конюшни. |
| `js/training.js` | Экран Тренировки. |
| `js/breeding-service.js` | Племенной жеребец/кобыла, цены, доступность, поиск родителей. |
| `js/breeding-logic.js` | Наследование, прогноз потомка, создание жеребёнка. |
| `js/breeding.js` | Координатор Разведения: выбранные родители, шаги, события, вызовы service/logic/render. |
| `js/breeding-render.js` | Визуальная подача Разведения: экран рождения, анимации, имя. Позже — весь HTML-рендер Разведения. |
| `js/race-menu.js` | Выбор гонки и лошади. |
| `js/race-engine.js` | Phaser-гонка. |
| `js/results.js` | Результаты гонки и награды. |
| `js/utils.js` | Общие утилиты без состояния. |

## Жёсткие правила

1. Не добавлять hotfix-файлы.
2. Не писать бизнес-логику в `index.html`.
3. Не дублировать карточку лошади. Использовать `SKACHKI_HORSE_UI.renderHorseCard`.
4. Не дублировать сервисных родителей вне `breeding-service.js`.
5. Не дублировать расчёты наследования вне `breeding-logic.js`.
6. Постоянные стили — только в CSS, не через JS `<style>`.
7. Новые файлы создавать только если они убирают смешение ответственности.
8. После изменения структуры обновлять этот документ.

## Карточка лошади

- Единый рендер: `SKACHKI_HORSE_UI.renderHorseCard`.
- Потенциал: звёзды.
  - 50–60: 1
  - 60–70: 2
  - 70–80: 3
  - 80–90: 4
  - 90–100: 5
- Класс: цифра на медальоне, без слова `Класс`.
- Пол: значок.
- Одна и та же карточка используется в Конюшне, Гонках, Разведении и на экране рождения.
- Экран рождения отличается только анимацией и подачей.

## Разведение

- Пользователь выбирает жеребца и кобылу.
- Если своих доступных жеребцов нет — доступен `Племенной жеребец` за `80`.
- Если своих доступных кобыл нет — доступна `Племенная кобыла` за `100`.
- Сервисные родители:
  - не сохраняются в Конюшню;
  - не получают `offspringCount`;
  - используются только для расчёта потомка.
- Потомок наследует породу, масть, характер, показатели, скрытые качества и потенциал.
- `Будущий жеребёнок` показывает прогноз, не точные значения.
- Точные значения открываются после рождения.
- Имя жеребёнка можно изменить на экране рождения.

## Telegram Mini App

- Верх не должен перекрываться интерфейсом Telegram.
- Основная навигация — нижнее меню.
- Важные кнопки не должны попадать под нижнее меню.
- Длинные экраны должны скроллиться.

## Рефакторинг Разведения

Текущий порядок:

1. `breeding-service.js` — сделано.
2. `breeding-logic.js` — создан и подключён.
3. Перевести `breeding.js` на `SKACHKI_BREEDING_LOGIC` — требуется замена файла.
4. Перенести HTML-рендер из `breeding.js` в `breeding-render.js`.
5. Оставить `breeding.js` только координатором.

## Проверка после изменений

После правок Разведения проверить:

- экран открывается;
- доступны жеребец и кобыла;
- работают племенные родители;
- цена считается корректно;
- кнопка `Скрестить` активируется корректно;
- жеребёнок создаётся;
- имя редактируется;
- жеребёнок появляется в Конюшне;
- сервисные родители не появляются в Конюшне;
- Конюшня, Гонки и Рейтинг не ломаются.
